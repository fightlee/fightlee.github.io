javascript高级(四)

作用域

​ 闭包的形成与变量的作用域以及变量的生命周期密切相关.

变量的作用域

​ 变量的作用域,指变量的有效作用范围.

  • 在JavaScript中,如果变量前没有加上var关键字,那么此时的变量为全局变量,这种命名方式很不安全,尽量避免使用.
  • 我们可以在函数内部对变量使用var进行声明,此时的变量为局部变量,即当前的变量的作用域只能在函数内部使用,函数外部使用无效.这里和Java其实是一样的.

变量的生命周期

​ 生命周期 : 一个对象从创建到销毁的过程.

  • 全局变量 : 除非主动销毁这个变量,不然就是其生命周期就是永久的
  • 局部变量 : 当函数执行完退出后,变量就会被释放.

编译原理(了解)

  • 分词/词法分析

    如果在一个javascript源码中书写了var a = 2; , 那么javascript会将其分解成以下这些词法单元(Token) : var, a , = , 2 ,; . 空格是否为合法的词法单元,取决于空格在这门语言中是否有意义,这个分解的过程就称之为分词过程.

    例

  • 解析/语法分析

    紧接上一个过程,将词法单元流(数组)转换成一个由元素逐级嵌套所组成的代表程序语法结构的树.这个数就称之为”抽象语法树“(Abstract Syntax Tree , AST) . AST以树状的形式表现语言的语法结构,树上的每个节点都表示源代码中的一种结构.AST树

    比如 : 5 + (10 * 2) 生成的AST树

  • 代码生成

    将AST树转换成可执行的代码的过程就被称为代码生成了.这个过程与语言,目标平台等息息相关.简单来说就是通过某种方法将源码通过语法分析后的AST树转换成一组机器指令,接着创建变量a,分配内存等操作,最后给变量a赋值.

执行过程(了解)

​ var a = 2;的执行过程主要分为2个阶段 : 这2个阶段涉及到三个对象

  • JavaScript引擎 : 如果把JavaScript程序当成一个公司,那么JavaScript引擎就是整个公司的Boss,负责整个JavaScript程序的编译及执行过程.
  • 编译器 : 编译器就是公司里Boss的最佳员工,所有公司的脏活累活全都交给他来干,比如(词法分析啊,生成AST树啊,代码生成什么的.)
  • 作用域 : 可以将作用域想象成另外一个员工,它负责收集并维护由所有声明的标识符(变量)组成的一系列查询,并实施一套非常严格的规则,确保当前执行的代码对这些标识符的访问权限,我们可以认为它是公司的安全员.这里还要注意,公司中可能有不同部门,部门中还有其他部门,那么每个部门都有其部门所在的安全员.
  1. 当编译器,我们的最佳员工碰到var a, 最佳员工就会去找作用域安全员,问他 : “喂,伙计,你这里有没有名称叫a的变量啊?”如果有,那么最佳员工就会忽略该声明,继续编译的干活,如果没有,就会叫安全员在当前他所在的部门中声明一个新的变量,名字叫做a.
  2. 在Boss开始干活时,也会问安全员: “喂,你这部门有没有a这个变量啊?”如果有,Boss就会开始使用这个变量a,如果没有Boss,就会继续问其他安全员,直到有一个安全员记录,并收集了a变量,就将2赋值给变量a,如果没找到,Boss反手就是一巴掌,并给我们抛出一个异常,=-=!!

查询过程(了解)

​ JavaScript中的Boss(引擎)执行代码时,会通过查找变量a来判断是否声明过,查找过程需要不断的询问作用域,也就是我们的安全员.在查找的过程中,Boss会通过2种方式来进行查询.

  • LHS(LEFT-HAND-SIDE) : 赋值操作的左侧 , 查找变量本身,就是为了给变量赋值.

  • RHS(RIGHT-HAND-SIDE) : 赋值操作的右侧 , 可以看成是(retrieve his source value) 获取它的源值,就是查找该变量的值,为了使用其中的值.

    当变量出现在赋值操作的左边时,进行LHS查询,出现在右侧进行RHS查询.

分析以下代码的查询过程 :

  1. 要执行foo()函数,所以对foo()函数进行RHS查询
  2. foo()函数中参数a的赋值,先查询a在作用域中是否存在,这里进行LHS查询,然后将2赋值给a.
  3. 要执行console.log()方法,首先对console对象进行赋值操作,进行RHS查询,然后检查log()函数在对象中是否存在
  4. 对log()方法中的参数a进行赋值操作,所以对a进行RHS查询,然后将2赋值给a.

作用域嵌套

​ 前文说过,在一个公司中,可能包含很多部门,在部门下还有其他部门.那么在程序中也如此,在程序的运行过程中,通常要顾及到几个作用域.

​ 当函数嵌套在另一个函数中时,就叫作用域的嵌套.如果引擎在当前作用域下没找到该变量,那么就会继续向外层嵌套的作用域去查找,直到最外层的作用域(全局作用域)为止,如果还没找到则抛出异常.

​ 我们可以将此过程想象成以下场景 : 在这个查询过程中,LHS和RHS都会向当前层级进行搜寻,如果没有找到就往外层寻找,直到最外层全局作用域.

​ 那么既然LHS和RHS都会向外层查找,那么为什么还要区分LHS和RHS?

​ 因为2者在没有找到变量时,所做的操作是不一样的.

  • 在执行RHS查询时在整个作用域都没有找到要赋值的变量.那么就会抛出异常.
  • 在执行LHS查询时,如果在最顶层作用域(全局作用域)都没找到该变量,我们的全局作用域就会帮助我们创建一个具有该名称的变量,并交给引擎运行,如果在严格模式下此操作无效.

小结

  • LHS查询,目的要找到该变量进行赋值操作.
  • RHS查询,目的为获取该变量的值.

词法作用域&动态作用域

​ 词法作用域是由我们在写代码时将变量和块作用域写在哪里来决定的,词法分析器处理代码时会保持作用域不变.

​ 动态作用域不关系函数和作用域是如何声明以及在何处声明,只关心他们从何处调用.

​ JavaScript并不具备动态作用域,这里只是为了拿出来和词法作用域做相对理解.

看下面例子: 分析词法作用域和动态作用域的结果.

  • 由于词法作用域只跟声明的函数和块的位置有关,跟调用的位置无关,所以当在foo函数中使用console.log()中使用变量a时,就会在当前作用域下寻找有没有a变量,如果没有,继续往上层找,此时在全局作用域找到var a = 2;所以输出2
  • 动态作用域跟调用的位置有关,那么我们从函数调用的位置开始寻找,首先还是从函数foo内部寻找,没找到从调用foo函数的bar函数内部找,此时找到var a = 3,那么就输出3

词法欺骗

​ 在程序运行过程中”修改”(欺骗)词法作用域的行为.

eval

​ 该函数用来执行参数中的带JavaScript代码的字符串内容

​ 注意 : 在严格模式下,eval()在运行时也有其自己的词法作用域,所以其声明不能修改所在的作用域.

with
with 语句用于设置代码在特定对象中的作用域。

​ with将一个没有或有多个属性的对象处理为一个完全隔离的词法作用域,因此这个对象的属性也会被处理为定义在这个作用域中的词法标识符.

​ 简单来说 , eval()函数中的参数中有一个或者多个声明的代码,那么就会修改其所处的词法作用域,而with就会根据传入的对象凭空创建一个全新的词法作用域.

​ 也就是说在此处,o1对象进入到foo函数内部传递到with语句中,with所声明的作用域就是o1,那么在o1作用域中有一个a属性的标识符,就做一个LHS引用给a属性赋值.同样的,o2对象传进去后,此时with语句的词法作用域为o2对象,那么o2作用域中没有找到a属性的定义,所有a的值为undefined,但是根据LHS引用的规则,如果在当前作用域下没有找到,就会向上继续寻找,如果到全局作用域下还没找到时,就会帮助我们创建一个全局变量a =2.所以紧接着在下方输出console.log(a);//2

小结

​ 虽然eval()和with都能使Javascript完成更为复杂的操作,但是由于性能和严格模式的原因,现在基本已经完全被禁用.

函数作用域

​ 这个函数中的全部变量都可以在整个函数的范围内使用及复用.

​ 函数作用域的作用 :

  • 用来封装要实现的功能逻辑
  • 用来隐藏某些变量或者函数(相当于Java私有化,封装的特性)

函数表达式和函数声明

声明函数

​ 声明函数可以用来隐藏某些代码和函数的实现,但是会污染当前所在作用域,并且需显示调用才能运行.

​ 解决”污染”作用域的方法,使用IIFE.

匿名函数表达式

​ 缺点

  • 不易调试
  • 函数需要引用自身需要使用过时的arguments.callee.
  • 可读性差
具名函数表达式

​ 区别函数声明和函数表达式,就看当前函数所在位置的第一个词是function就是声明,反之为函数表达式.

IIFE

​ IIFE : (Immediately Invoked Function Expression) 立即执行函数.

​ 函数表达式语法 : 将函数包裹在()内部当成表达式,再使用()调用该函数表达式

​ 作用 : 可以当成函数调用,并传递内部运行时需要的参数.
