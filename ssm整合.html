总结

  • Mybatis 虽然好用,但是对于一些基本的 CRUD 的操作仍需自己编写 Service 接口,Mapper 接口和其具体实现

使用 Mybatis-plus

修改 pom.xml

  • 删掉 mybatis 原有的依赖和 mybatis-spring 的依赖,添加 mybatis-plus 依赖(避免冲突)

修改 Dao 层

  • UserMapper.class继承BaseMapper.class接口即可,泛型使用当前操作的 Model 类型,基本的 CRUD 都已经在 BaseMapper 中实现,不用写 XXMapper.xml 文件和定义 CRUD 方法.

修改 Service 层

  • mybatis-plus 提供了 IService 接口,接口中同样提供了大量的 CRUD 业务方法.直接创建 XXService 实现该接口即可

修改 Model 层

  • 这里使用 Lombok 自动生成 JavaBean 的构造
    • @AllArgsConstructor : 所有参数构造
    • @NoArgsConstructor : 无参构造
    • @Getter : 所有属性 getter 方法
    • @Setter : 所有属性 Setter 方法
    • @ToString : 所有属性 toString 方法
    • @TableName : 该注解为 Mybatis-plus 提供注解,当 Bean 的名称和表名不一致时,需要添加该注解,注解中的值为数据库中所对应的表名.
