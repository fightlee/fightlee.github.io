4) 和 UserMapper.xml 同一文件夹中创建 UserMapper.java

com.java.dao.UserMapper.class

5) test 包下创建测试类

注意

  1. 主配置文件中的 mapper 扫描方式建议换成package扫描.可以同时扫描当前包下的所有接口和 xml 文件
  2. 如果使用package扫描要注意以下几点 :
    1. XXMapper.java 接口文件的路径要和 XXMapper.xml 文件的 namespace 要一致
    2. XXMapper.xml 的配置文件中的 select 等标签的 id 要和 java 接口文件对应的方法名称一致
    3. 配置文件的 select 等标签的返回值要和 java 接口文件对应的方法的返回值一致

如果以上配置都没有问题,还会出现 XXMapper.xml 文件找不到或者主配置文件找不到的情况.进行以下操作.

  1. 在 pom.xml 中添加插件,重新 reload maven
  1. 在上方菜单栏中找到Build ===> Rebuild Project

  1. 在 maven 命令中先执行clean,然后重新compile.

  1. 还不行,直接复制文件到target目录中

  1. 还不行,就上面的步骤多试几次.

  2. 还不行,抽个烟冷静下吧.再不行电脑都给他砸咯.

级联映射
  • result中的column为查询语句中查询出的列名,如果有别名使用别名,property为当前属性所属类的属性名
  • javaType为关联属性类型
关联映射
  • 使用association标签来关联当前对象中的1对1对象属性level,关联的属性column仍然为查询出的列名, property为 Level对象的属性名
  • 缺点 : 不能使用懒加载
子查询
  • 使用association标签中的select属性来标识使用Level命名空间下的方法来查询Level的属性值.
  • javaType为关联属性类型
  • 通过column属性中的列名将查询出的Level的id传递给子查询语句,注意,这里的名称仍然是查询出的列名.
  • 可以通过打开Mybatis-config的配置来使用对象延迟加载.使用到Level对象才会发送SQL语句查询Level对象信息.

Mybatis 缓存

概念 : 使用 Mybatis 查询相同数据时,根据使用的缓存级别不同来缓存该数据对象.下一次查询相同数据时,直接从缓存中查询数据,不需要再次访问数据库.

一级缓存

  • session 级别缓存
  • 不同 session 缓存失效
  • 查询同一数据之间进行了增删改的操作,缓存失效
  • 默认开启,不需要设置

二级缓存

  • namespace 级别缓存

  • 不同 session 缓存依旧生效

  • 需要在对应的XXMapper.xml文件中添加cache标签

    • eviction : 使用的缓存策略
    • readOnly : 是否直接缓存对象引用
    • size : 缓存大小
    • type : 指定第三方缓存的类文件
